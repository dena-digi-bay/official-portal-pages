# official-portal-pages

本仓库用于归档和发布多个独立 HTML 页面。这些页面可作为门户、引导页、信息展示页等用途，不绑定任何特定域名或网站。

## 项目简介

本项目旨在收集、整理并托管一组轻量级 HTML 页面。每个页面均为独立文件，无复杂依赖，便于直接部署或二次修改。仓库内容适合需要快速搭建静态入口页面的场景。

## 目录说明

```
official-portal-pages/
├── pages/          # 存放所有 HTML 页面文件
├── assets/         # 页面引用的静态资源（CSS、JS、图片等）
├── docs/           # 项目相关文档或说明
└── README.md       # 本文件
```

- `pages/`：主要页面目录，每个 `.html` 文件对应一个独立页面。
- `assets/`：页面共享或专用的资源文件，按需组织。
- `docs/`：附加说明、设计稿或规范文档。

## 页面归档说明

- 所有页面均为纯前端静态 HTML，无后端依赖。
- 页面之间彼此独立，可单独部署到任意静态托管服务。
- 文件命名建议使用小写英文加连字符，例如 `welcome-page.html`。
- 归档时请保持文件结构清晰，避免深层嵌套。

## 维护说明

- 欢迎提交新页面或改进现有页面，请遵循以下基本规范：
  - 页面需包含完整的 HTML 结构（`<!DOCTYPE html>`、`<head>`、`<body>`）。
  - 避免使用外部不可控资源（如第三方 CDN 中的动态内容）。
  - 资源文件尽量放入 `assets/` 对应子目录。
- 提交 Pull Request 时请简要说明页面用途或修改内容。
- 如有页面废弃或需要移除，请注明原因并在合并前清理引用。
